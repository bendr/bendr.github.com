Bender

A Declarative, Dynamic Framework for Web Application User Interfaces

Julien Quint, IGEL Co., Ltd.

Version 0.8, 2013-01-25

1 Introduction

This specification defines the XML syntax, Javascript API, and informal semantics of the Bender framework. Bender is an experimental framework aimed at Web application user interface authoring. It is declarative, building on existing declarative languages such as HTML, SVG and CSS to describe the layout, structure as well as the logic of a Web application. A Bender application description can be serialized to XML following the syntax described in this specification. Such a description can then be read and interpreted by a runtime inside a host application such as a Web browser.

2 Document structure

A Bender XML document always describes a component; therefore, the document element is always a component.

The Bender namespace URI

All Bender elements are in the following namespace URI:

http://bender.igel.co.jp

3 Elements

3.1 The component element

Define a Bender component.

Child of:

  • component
  • content
  • foreign content
  • view

Content:

  • optional delegate attribute
  • optional href attribute
  • optional id attribute
  • zero or more other attribute
  • interleaving of:
    • zero or more component element
    • zero or more link element
    • zero or more property element
    • optional view element
    • zero or more watch element

The component is the core element in Bender.

3.2 The content element

Provide a location for custom rendering for derived components.

Child of:

  • foreign content
  • view

Content:

  • mixed content:
    • zero or more component element
    • zero or more foreign content element

3.3 Foreign content

Foreign content appearing in the view of a component.

Child of:

  • content
  • foreign content
  • view

Content:

  • zero or more free attribute
  • mixed content:
    • zero or more component element
    • optional content element
    • zero or more foreign content element

3.4 The get element

Custom getter for a property.

Child of:

  • property

Content:

  • choice of:
    • value attribute
    • text

3.5 The get element

Event or property being watched.

Child of:

  • watch

Content:

  • choice of:
    • group of:
      • event attribute
      • optional component attribute
    • group of:
      • dom-event attribute
      • optional view attribute
    • group of:
      • property attribute
      • optional component attribute
  • optional choice of:
    • value attribute
    • text

3.6 The link element

Associate external resources with the component.

Child of:

  • component

Content:

  • rel attribute
  • href attribute

This element is similar to the HTML link element. It allows to link the component to other resources, namely stylesheets and scripts.

The destination of the link is given by the href attribute, which must be present and must contain a valid non-empty URL potentially surrounded by spaces.

The type of link is indicated by the rel attribute, which must be present and must contain a valid keyword potentially surrounded by spaces. The valid keywords and their signification are:

  • script: import a script. The URL must point to a Javascript file.
  • stylesheet: import a stylesheet. The URL must point to a CSS file. Handling of stylesheet is dependent on the runtime: in the case of HTML output, a new link element is created in the head of the document; in the case of SVG, a new link element is created in the svg element of the document (using the correct xlink:href attribute.)

The context manages the linked resources for all components and loads resources only once.

3.7 The property element

Define a property of the component.

Child of:

  • component

Content:

  • name attribute
  • optional as attribute
  • optional value attribute
  • interleaving of:
    • optional get element
    • optional set element

3.8 The set element

Custom setter for a property.

Child of:

  • property

Content:

  • choice of:
    • value attribute
    • text

3.9 The set element

Action associated with a watch.

Child of:

  • watch

Content:

  • choice of:
    • group of:
      • attr attribute
      • optional view attribute
    • group of:
      • event attribute
      • optional component attribute
    • group of:
      • property attribute
      • optional component attribute
  • optional choice of:
    • value attribute
    • text

3.10 The view element

Describe the rendering of a component.

Child of:

  • component

Content:

  • mixed content:
    • zero or more component element
    • optional content element
    • zero or more foreign content element

3.11 The watch element

Define a watch for the component.

Child of:

  • component

Content:

  • interleaving of:
    • zero or more get element
    • zero or more set element
